# Atoms Demo 使用说明

[English](README.md) | 中文使用说明

## 项目简介

Atoms Demo 是一个 AI 驱动的代码生成和可视化 Web 开发平台，灵感来自 [Atoms](https://atoms.dev/)。你可以用自然语言描述需求，AI 会帮你生成完整的可运行 HTML 应用，并提供实时预览。

## 功能特性

- 👤 **用户认证**：邮箱/密码注册登录
- 📁 **项目管理**：创建和管理多个项目
- 🤖 **AI 代码生成**：通过文本提示生成完整的 HTML 应用
- 👁️ **实时预览**：生成代码即时预览
- 📝 **内置编辑器**：集成 Monaco Editor 支持手动编辑
- 🧩 **组件库**：保存生成的组件，在不同项目中复用
- 💾 **数据持久化**：开发环境使用 SQLite，生产支持 PostgreSQL

## 技术栈

- **前端**：React 18 + TypeScript + Vite + Tailwind CSS
- **后端**：Node.js + Express
- **数据库**：SQLite / PostgreSQL + Prisma ORM
- **AI**：兼容 OpenAI 格式的 API（OpenAI、火山方舟等）
- **编辑器**：Monaco Editor

## 快速开始

### 1. 安装依赖

```bash
npm run install:all
```

### 2. 配置环境变量

复制环境变量示例文件：

```bash
cp .env.example .env
```

编辑 `.env` 文件，根据你的 AI 提供商选择一种配置：

#### 方式一：使用 OpenAI

取消注释并填写：

```env
OPENAI_API_KEY="your-openai-api-key-here"
OPENAI_BASE_URL="https://api.openai.com/v1"
AI_MODEL="gpt-4o"
```

> 注意：`OPENAI_API_KEY` 从 [OpenAI 官网](https://platform.openai.com/api-keys) 获取

#### 方式二：使用火山方舟（ByteDance）

取消注释并填写：

```env
VOLC_API_KEY="your-volcengine-ark-api-key-here"
VOLC_BASE_URL="https://ark.cn-beijing.volces.com/api/v3"
AI_MODEL="deepseek-ai/deepseek-coder-250k"
```

> 注意：
> - `VOLC_API_KEY` 在火山方舟控制台获取
> - `AI_MODEL` 需要填写**模型接入点 ID**，不是模型名称

**常见问题**：
> `404 The model or endpoint minimax-m2.5 does not exist`
>
> 原因：你直接使用了模型名称 `minimax-m2.5`，但火山方舟要求填入**接入点 ID**。
>
> 解决方法：在火山方舟创建模型接入点后，使用完整的接入点 ID，例如 `minimax/minimax-m2-5-pro-128k` 或你创建后得到的接入点 ID。

#### 方式三：使用其他兼容 OpenAI 格式的 API

配置以下三个变量：

```env
OPENAI_API_KEY="your-api-key"
OPENAI_BASE_URL="https://your-api-base-url/v1"
AI_MODEL="model-name"
```

### 3. 初始化数据库

```bash
npm run prisma:migrate
```

这会创建 SQLite 数据库表结构。

### 4. 启动开发服务器

```bash
npm run dev
```

启动成功后：
- 前端访问：http://localhost:5173
- 后端 API：http://localhost:3001

## 使用流程

1. **注册/登录**：创建一个账号或登录已有账号
2. **新建项目**：点击创建新项目，用自然语言描述你想要构建的应用
3. **AI 生成**：AI 会生成完整的 HTML 代码（包含 HTML + CSS + JS）
4. **实时预览**：立即在右侧看到生成应用的预览效果
5. **编辑修改**：如果需要，可以直接在内置编辑器中手动修改代码
6. **迭代优化**：通过新的提示要求 AI 进行额外修改
7. **组件复用**：将有用的组件保存到组件库，以后可以重复使用

## 详细功能操作

### 👤 用户注册与登录

#### 注册新账号
1. 打开 http://localhost:5173
2. 点击「注册」标签
3. 输入你的邮箱和密码
4. 点击「注册」按钮完成注册
5. 注册成功后会自动登录

#### 登录
1. 点击「登录」标签
2. 输入邮箱和密码
3. 点击「登录」按钮

> 提示：数据保存在本地 SQLite 数据库中，账号信息不会上传到第三方服务器。

---

### 📁 项目管理

#### 新建项目
1. 登录后进入项目列表页面
2. 点击右上角「新建项目」按钮
3. 输入项目名称
4. 在描述框中用自然语言描述你想要创建的应用，例如：
   ```
   一个漂亮的计算器，支持加减乘除，深色主题
   ```
   或者：
   ```
   一个待办事项列表，可以添加、删除、标记完成
   ```
5. 点击「创建并生成」按钮
6. 等待 AI 生成代码，完成后会自动跳转到项目编辑器

#### 打开已有项目
- 在项目列表页面点击任意项目即可打开

#### 删除项目
1. 在项目列表找到要删除的项目
2. 点击项目卡片右下角的删除图标
3. 确认删除后项目会被永久删除

---

### 🤖 AI 代码生成

#### 首次生成
进入项目后，AI 会自动根据你的描述生成代码。生成完成后：
- 左侧显示代码编辑器
- 右侧实时显示预览效果

#### 迭代修改
如果你对生成结果不满意，想要进一步修改：
1. 在页面底部的「发送提示」输入框中输入你的修改要求，例如：
   ```
   添加一个清空按钮，把背景改成浅蓝色
   ```
2. 点击「发送」按钮
3. AI 会基于当前代码进行修改，生成新的完整代码

> 提示：每次修改都会替换整个代码，确保一致性。

---

### 📝 代码编辑

#### 手动编辑代码
- 直接在左侧 Monaco Editor 中修改 HTML 代码
- 修改后预览会自动更新
- 支持语法高亮、代码补全

#### 保存修改
- 点击编辑器上方的「保存」按钮
- 修改会自动保存到数据库

#### 切换预览/编辑模式
- 点击「预览」可以只显示预览，隐藏编辑器
- 点击「编辑」可以回到编辑模式

---

### 🧩 组件库使用

#### 什么是组件库？
组件库允许你保存经常复用的 HTML 代码片段（比如导航栏、卡片、表单等），在不同项目中重复使用。

#### 新建组件（保存当前项目代码为组件）
方法一：从当前项目保存
1. 打开你想要保存组件的项目
2. 确保当前代码是你想要保存的版本
3. 点击左侧边栏的「组件库」图标
4. 点击组件库右上角的「+ 新建组件」按钮
5. 输入组件名称（例如「响应式导航栏」）
6. 输入组件描述（可选，说明这个组件的用途）
7. 点击「创建」即可保存

方法二：从零新建
1. 进入组件库页面
2. 点击「+ 新建组件」
3. 填写名称和描述
4. 在代码编辑器中输入组件 HTML 代码
5. 点击「创建」保存

#### 使用组件（插入到当前项目）
1. 打开你要编辑的项目
2. 点击左侧边栏的「组件库」图标
3. 在组件列表中找到你想要插入的组件
4. 点击「插入」按钮
5. 组件代码会被插入到当前项目的代码中
6. 记得点击「保存」保存修改

> 提示：插入组件后，你可能需要调整一下代码位置，把组件放到合适的地方。

#### 编辑已有组件
1. 在组件库中点击要编辑的组件
2. 修改代码或名称/描述
3. 点击「保存」更新

#### 删除组件
1. 在组件库中找到要删除的组件
2. 点击删除按钮
3. 确认删除后组件会被永久删除

> 注意：删除组件不会影响已经使用了该组件的项目，因为代码已经插入到项目中了。

### 💡 使用提示与技巧

1. **描述越清晰，生成效果越好**
   - 不好：`做一个博客`
   - 好：`一个个人博客首页，包含文章列表、侧边栏、分页，使用响应式设计`

2. **分步构建复杂应用**
   - 先生成基础结构
   - 再逐步添加功能
   - 每次迭代只改一个部分

3. **善用组件库**
   - 把常用的页面组件（导航、页脚、表单）保存下来
   - 下次新建项目直接插入，节省时间

4. **AI 生成后手动微调**
   - AI 生成的代码可以作为基础
   - 手动调整样式和细节，达到理想效果

## 📋 完整使用示例：创建一个待办事项清单

我们从零开始，一步步创建一个可交互的待办事项清单应用。

### 第一步：启动服务
确保你已经完成了环境配置，启动项目：
```bash
npm run dev
```

打开浏览器访问：**http://localhost:5173**

### 第二步：注册账号
1. 你会看到登录/注册页面
2. 点击「注册」标签
3. 输入：
   - 邮箱：`test@example.com`
   - 密码：`123456`
4. 点击「注册」按钮
5. 注册成功后自动登录，进入项目列表页面

### 第三步：创建新项目
1. 在项目列表页面，点击右上角**「新建项目」**蓝色按钮
2. 填写：
   - **项目名称**：`我的待办清单`
   - **项目描述**：
     ```
     做一个美观的待办事项清单应用，要求：
     - 可以添加新的待办事项
     - 可以标记待办事项为已完成
     - 可以删除待办事项
     - 显示未完成/已完成统计
     - 使用现代化的 UI，圆角卡片，柔和颜色
     - 响应式设计，手机上也能正常用
     ```
3. 点击**「创建并生成」**

### 第四步：等待 AI 生成
- 页面会显示"正在生成代码..."
- 等待十几秒到几十秒（取决于 AI 响应速度）
- 生成完成后自动跳转到编辑器页面

### 第五步：查看结果
生成完成后，你会看到：
- **左侧**：AI 生成的完整 HTML 代码（包含 HTML + CSS + JavaScript）
- **右侧**：已经渲染好的待办清单应用，可以直接交互使用

现在你就可以直接在右侧预览里试用：添加待办、勾选完成、删除项。

### 第六步：修改需求（迭代优化）
如果你觉得不好看，想要改成深色主题：
1. 找到页面底部的**「发送提示」**输入框
2. 输入：
   ```
   把整体改成深色主题，背景用深色，文字用浅色
   ```
3. 点击**「发送」**
4. AI 会重新生成代码，几秒钟后更新完成
5. 右侧预览自动刷新，现在就是深色主题了

### 第七步：保存代码为组件（供以后复用）
如果你觉得这个待办组件写得很好，想保存到组件库以后用：

1. 点击左侧边栏最下方的**🧩 组件库**图标
2. 点击组件库面板右上角的**「+ 新建组件」**
3. 填写：
   - **组件名称**：`深色主题待办清单`
   - **组件描述**：支持增删改查的深色风格待办组件
4. 点击**「创建」**
5. 组件就保存好了！

### 第八步：在新项目中使用已保存的组件
1. 回到项目列表，新建一个项目"我的主页"
2. 项目描述写：`一个个人主页，包含导航栏、关于我、待办事项区域`
3. 创建生成后，打开项目编辑器
4. 点击左侧边栏的**🧩 组件库**图标
5. 找到刚才保存的`深色主题待办清单`
6. 点击**「插入」**
7. 组件代码就插入到当前项目了
8. 在编辑器里微调一下代码位置，放到你想放的地方
9. 点击「保存」

## 常见问题排查

### Q1: 报错 404 The model does not exist or you do not have access to it

**原因**：`AI_MODEL` 配置不正确。

**解决**：
- **OpenAI**：确认模型名称正确，比如 `gpt-4o`、`gpt-4o-mini`
- **火山方舟**：必须使用**接入点 ID**（格式：`developer-name/model-name`），不是裸模型名。在火山方舟控制台 -> 模型接入点 查看你的接入点 ID。

**正确配置示例（火山方舟）**：
```env
# 正确
AI_MODEL="deepseek-ai/deepseek-coder-250k"
AI_MODEL="minimax/minimax-m2-5-pro-128k"

# 错误（会报 404）
AI_MODEL="minimax-m2.5"
```

### Q2: 前端可以访问，但 AI 生成失败

**检查**：
1. 确认 API Key 正确且有余额
2. 确认 `VOLC_BASE_URL` 或 `OPENAI_BASE_URL` 配置正确
3. 检查后端日志：`npm run dev` 输出会显示具体错误

### Q3: `npm run install:all` 失败怎么办

手动分步执行：
```bash
# 安装根目录依赖
npm install

# 安装前端依赖
cd frontend && npm install && cd ..

# 安装后端依赖
cd backend && npm install && cd ..
```

### Q4: 数据库迁移失败

```bash
# 重新执行迁移
npx prisma migrate dev --name init
```

## 可用的 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/register` | 用户注册 |
| POST | `/api/auth/login` | 用户登录 |
| GET | `/api/auth/me` | 获取当前用户信息 |
| GET | `/api/projects` | 获取用户所有项目 |
| POST | `/api/projects` | 创建新项目 |

| POST | `/api/generation` | AI 生成代码 |
| GET | `/api/components` | 获取用户所有组件 |
| POST | `/api/components` | 创建组件 |
| DELETE | `/api/components/:id` | 删除组件 |
